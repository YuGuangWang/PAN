<p>We compiled this checklist by looking at what's common to the most popular ML research repositories. In addition, we prioritized items that facilitate reproducibility and make it easier for others build upon research code.</p>
<p>The ML Code Completness Checklist consists of five items:</p>
<ol>
<li><strong>Specification of dependencies</strong></li>
<li><strong>Training code</strong></li>
<li><strong>Evaluation code</strong></li>
<li><strong>Pre-trained models</strong></li>
<li><strong>README file including table of results accompanied by precise commands to run/produce those results</strong></li>
</ol>
<p>We verified that repositories that check more items on the checklist also tend to have a higher number of GitHub stars. Pre-trained models</h4>
<p>Training a model from scratch can be time-consuming and expensive. One way to increase trust in your results is to provide a pre-trained model that the community can evaluate to obtain the end results. This means users can see the results are credible without having to train afresh.</p>
<p>Another common use case is fine-tuning for downstream task, where it's useful to release a pretrained model so others can build on it for application to their own datasets.</p>
<p>Lastly, some users might want to try out your model to see if it works on some example data. Providing pre-trained models allows your users to play around with your work and aids understanding of the paper's achievements.</p>
